---
summary: "Self-managing context architecture for AI-assisted coding. Gives an AI coding agent the right context at the right time — without flooding the context window."
---

dev-session is a TypeScript npm package that solves a real problem in AI-assisted development: how do you give a coding agent the right context without overwhelming it?

The current approach — dumping entire codebases into a context window — is a blunt instrument. dev-session introduces selective file-layer loading via JSDoc-style annotations. Layers are declared, not dumped. The package manages its own context lifecycle.

v2 introduces a more structured layer model: files annotate which context layer they belong to, and the agent loads only the layers relevant to the current task. This is context engineering applied as architecture rather than as prompt hacking.

The project is a direct product of thinking seriously about how agentic coding tools work mechanically — the ReAct loop, subagent spawning, token budgets, and the semantics of a well-designed AGENTS.md file. Understanding the internals changes how you use the tool.
